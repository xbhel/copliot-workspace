# Quick Start

Create a concise, beginner-friendly quick-start guide for any topic the user wants to learn.

The guide should cover:

1. Overview — what is this topic about and why is it useful?
2. Prerequisites — what do users need to know or have before getting started with this topic?
3. Getting Started — what are the first steps to take to start using or learning about this topic?
4. Core Concepts — what are the most important things to understand about this topic?
5. Architectural Overview (if applicable) — high-level explanation of how the components of this topic fit together
6. Work Model (if applicable) — how do users typically interact with this topic in practice?
7. Real World Use Cases — examples of how this topic is used in real life
8. Common Pitfalls — what are common mistakes or misconceptions to avoid when learning about this topic?
9. Security (if applicable) — what are the key security considerations or best practices related to this topic?
10. Tips and Best Practices — what are the most important tips for beginners to know when getting started with this topic?
11. Common Pitfalls — what are common mistakes or misconceptions to avoid when learning about this topic?
12. Next Steps — what should users do after reading this guide to continue learning or using this topic effectively?

Add any other helpful sections, in a logical order, when they are relevant to the topic and useful for beginners.

You must:

- Keep the guide concise and beginner-friendly
- Use clear, simple language and avoid jargon
- Keep warm and encouraging tone throughout the guide, never cold or clinical
- Add appropriate emojis sparingly to make the guide more engaging and visually appealing, without overusing them
- Add colored Mermaid diagrams or other visual aids when they help explain complex concepts more clearly. If helpful, include numbered steps or sequence markers in the diagram to make the flow easier to understand
- Avoid Mermaid text or labels that may be parsed as Markdown or other rich-text syntax; use plain text or safer alternatives instead, for example, "1: run task" instead of "1. run task"

Save the guide as `<topic>-quickstart.md` in the appropriate directory, using a well-structured format with clear headings and sections for each relevant point above.
